# ADR-003 — Partial Classes, 400-Line File Limit

## Status
Accepted

## Context
LLM agents (including GitHub Copilot) have output token limits.
Files over ~400 lines frequently cause incomplete edits, truncated output,
and agent errors when the agent attempts to read or rewrite them.

## Decision
All non-POCO classes in this project **must** be declared `partial`.

File size rules:
- Each `.cs` file must be **< 400 lines**.
- If a class exceeds 400 lines, its methods are split into separate files.
- File naming convention: `ClassName.FeatureName.cs`

### Example split
```
MessageDispatcher.cs              — class declaration, constructor, fields
MessageDispatcher.Routing.cs      — route/dispatch logic
MessageDispatcher.Lifecycle.cs    — init/dispose
```

## What counts as a POCO (exempt from partial requirement)
- Classes with only auto-properties and no methods.
- Record types.
- Enum types.

## Consequences
- Every class scaffold generated by the agent must include `partial`.
- Roslyn emission in ContinueTranslator must emit `partial` on all class declarations.
- Code review should reject non-partial non-POCO classes.
